<h1 align="center"> 📊 Retail Transaction Data Analysis & KPI Dashboard Development </h1>

## 📌 Project Overview

This project presents an end-to-end retail data analytics workflow designed to transform raw transactional data into actionable business insights.The objective was to analyze sales performance, customer behavior, product demand, and operational efficiency using SQL for data processing and Excel for dashboard development. The project demonstrates practical skills aligned with entry-level Data Analyst and Business Analyst roles.

## 🎯 Business Objectives

- Calculate total revenue and key performance indicators (KPIs)
- Identify top-performing locations and customer segments
- Analyze product demand and sales trends
- Evaluate payment method distribution
- Assess the impact of discounts on spending behavior
- Generate data-driven business recommendations

## 🗂 Dataset Description

The dataset contains retail transaction records with the following fields:

- Transaction ID
- Customer ID
- Customer Category
- Item
- Price Per Unit
- Quantity
- Total Spent
- Payment Method
- Location
- Transaction Date
- Discount Applied

## 📈 Dashboard Features

- KPI Cards (Total Revenue, Avg Transaction Value, Total Transactions)

- Discount vs Non-Discount Spending Comparison

## 💡 Key Insights

- Identified high-performing locations driving majority revenue.
- Determined customer segments contributing highest spending.
- Recognized top-selling products for inventory optimization.
- Evaluated whether discounts increase or dilute average transaction value.
- Highlighted opportunities for operational and marketing improvements.

## 📌 Business Recommendations

Focus marketing efforts on high-value customer categories.
Optimize stock levels for top-performing products.
Review discount strategy to balance revenue growth and profitability.
Implement targeted strategies for underperforming locations.
